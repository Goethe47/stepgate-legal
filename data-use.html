How We Use Data

Plain-language explanation of why StepGate needs certain permissions.

Last updated: 2026-05-10

Why app usage access is needed

StepGate reads usage data only to track time in selected controlled apps and detect when access runs out.

Why steps / activity are needed

StepGate can count steps with the phone sensor when Activity Recognition is granted.

Health Connect is optional. If you connect it, StepGate can use external step history; sync may still be delayed by source apps.

Why display over other apps is needed

StepGate uses display-over-apps permission only to show the limit screen over selected controlled apps. It does not read messages, screen text, keyboard input, or app content.

Does data leave the device?

No. StepGate stores this data locally and does not send usage/step data to servers.

What if permission is not granted?

StepGate still runs, but related features are limited. For example, without step data, step-based access updates are unavailable.

What StepGate does not access

  • Messages or chat contents
  • On-screen text in other apps
  • Keyboard input or passwords
  • Photos, files, or contacts
